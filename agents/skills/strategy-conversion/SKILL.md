# SKILL.md

## Name
strategy-conversion

## Used By
Mid Backend Engineer

## Purpose
Implement a strategy worker from an approved spec using existing platform patterns.

## Inputs
- approved strategy spec
- acknowledged mapping template
- reference implementation
- platform standards

## Outputs
- mapping document
- code changes
- tests
- PR description draft
- self-review checklist

## Procedure
1. Fill mapping document before code.
2. Confirm Senior Backend acknowledgement field is true before major implementation.
3. Implement using standard state machine and durable event rules.
4. Add tests first or alongside implementation.
5. Complete self-review checklist.
6. Link all artifacts in PR description.

## Stop Conditions
- mapping not acknowledged
- need for a missing platform component
- uncertainty about strategy logic not covered in spec
